Role Descriptor: Software Requirement Engineer
The Software Requirement Engineer elicits, analyses and documents Software Requirements into the Software Requirement Specification.

Main Description

The Software Requirement Engineer is responsible for gathering, analysing, documenting and maintaining the Software Requirements on which the project will be developed. Together with developers and testers he will discuss and negotiate with all stakeholders on the requirements, in order to develop the Software Requirement Specification.
The Software Requirement Engineer usually starts his work very early on in the project or even during the quotation phase. Once the Product Backlog starts taking shape, he will focus on the highest priority stories and assign requirements to them, as these stories will be next in line to be assigned to a Sprint. Therefor they need to have enough detail so the Scrum Team can start its work.
Throughout the project, the Software Requirement Engineer maintains and updates requirements, for example due to Software Change Request or Problem Reports.
