Main Description

1. Introduction
Is  a mechanism to generate a program to allow the code to be written at a higher abstraction level; Software Component Model. The code generation ensures that the Software Component Model is translated to a language the target or integrated development environment can underderstand.

2. Code Generation
Specific setting are needed to ensure an optimal code generation. This depends on the language of the generated code, the target on which the code will be used and also the specific functionality of the Software Component Model. Not all language specific addition can be added during the code generation and need to be applied afterwards. A static test will ensure that this process in performed accordingly.

Steps
Verify code generation settings
Verify if the code generation settings are according the defined settings.
Determine code generation optimisations
Depending on the type of functionality of the generated Software Component Model, code generation optimisations need to be determined.
Generate Code
Perform the actual generation of the code.
Apply additional updates
Apply the additional updates to the generated code if these cannot be integrated into the code generation. These updates can be done manually but it is preferred that this is done automatically to reduce the risks of making errors.
Ensure traceability
Establish traceability from the code to the Software Component Model.
Perform static test
Check to determine the sanity of the code. It is checking the code and/or manually reviewing the code to find errors.
