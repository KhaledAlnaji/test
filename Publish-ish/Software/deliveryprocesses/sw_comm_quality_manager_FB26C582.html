Role Descriptor: Software Quality Manager
The Software Quality Manager is responsible for monitoring and improveming software quality and software quality processes for all projects under his supervision.
Based on Method Role: Software Quality Manager
Relationships
Performs:
  • Develop Software Quality Assurance Plan
Responsible for:
  • Review Report
  • Software Quality Assurance Plan
  • Software Quality Assurance Strategy
  • Software Quality Report
Additionally Performs:
  • Develop Master Test Plan
  • Develop Software Change Management Plan
  • Develop Software Configuration Management Plan
  • Develop Software Development Plan
  • Perform Software Project Kick-off
Modifies:
  • Software Quality Assurance Plan
Main Description
The Software Quality Manager develops and maintains the software quality management strategy and planning for all projects under his supervision. He monitors the software quality for those projects, drives actions that lead to improvement of that quality and reports on the quality status of each project.
Additionally he coaches teams and team members in quality awareness and methods on how to improve software quality.
Properties
Multiple Occurrences
Optional
Planned
Key Considerations
The Software Quality Manager can not be under line management of the Software Project Leader or Scrum Master. He needs to have independence from the projects he is supervising.
