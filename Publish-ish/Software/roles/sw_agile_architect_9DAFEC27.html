Role: Software Architect
The Software Architect develops the Software Architecture Design, designating SW functionality to modules and defining their interfaces and dynamic behaviour.
Role Sets: All Process Roles, Scrum Team Roles
Relationships
Primary Performs
  • Develop Software Architecture Design
  • Perform Formal Review

  • Develop Software Requirement Specification
  • Investigate Change Request or Problem Report
  • Perform Project Quality Audit
  • Perform Software Integration
  • Perform Software Integration Test
  • Perform Software Project Kick-off
  • Raise Software Change Request or Problem Report
Modifies
  • Reviewed Work Product
  • Review Report
  • Software Architecture Design
  • Software Integration Test Specification
    Main Description

    The Software Architect develops and maintains the Software Architecture. He guards the overall consistency of the architecture and assigns requirements and functionality to architectural components.

    Since architecture is usually complex and spanning over the entire software structure, the Software Architect is always working on the architecture for the next Sprint. This prevents bottlenecks at the beginning of a Sprint.

    Staffing
    Skills

    1) Automotive domain knowledge
    2) AutoSAR knowledge
    3) Understanding of architectural design languages (SysML, UML, etc)
    4) Embedded System knowledge
