      Main Description
      Define a process by which changes to the software (due to new features or issues) are registred, assessed, and implemented in the software.
      In an Agile process, this may focus on the process the Product Owner uses to create new backlog items and to assess their priority. How these changes to the software are monitored through the implementation phase is described in more detail in the software quality assurance plan.
