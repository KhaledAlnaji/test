Task: Develop Software Development Plan
Creation of a comprehensive, composite document that gathers all information required to manage the project.
Disciplines: All Software Tasks, MAN.3 - Software Project Management
Purpose
Task to gather all of the information necessary to control the project. It describes the approach to the development of the software, and is the top-level plan generated and used by the managers to direct the development effort.

        Main Description

        1. Introduction
        The Software Development Plan is created during the Project Start-up phase and needs to be updated continuously. The Software Development Plan is a source containing the latest information regarding  the project and its process. Changes in information that are relevant for the development team shall be communicated during team meetings.

        2. Define Project
        Defining the project is the task to clearly describe the project objective and scope with their deliverables and milestones. Next to this all assumptions, dependencies and constraints of the project needs to be defined. These items will provide a better understanding why the project is setup a particular way and will aid in the decision making during the project.

        3.Define tooling
        After the project is defined the tooling and infrastructure can be selected based on the supporting processes needed for the project.
