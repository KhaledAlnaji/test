Task: Perform Software Project Kick-off
Alignment of all project members and customer at the start of the project.
Disciplines: All Software Tasks, MAN.3 - Software Project Management
Purpose
Define the base elements for the project and other project planning activities. Introduce the members of the project team and the client and provides the opportunity to discuss the role of each team member. Other base elements in the project that involve the client may also be discussed.

      Main Description

      The basic format of a kickoff is meeting including the roles of and contributions from the stakeholders. It also mentions the risks and issues involved, particularly those that may affect project schedules and lead to decreased customer satisfaction. This is the point at which help needed from stakeholders is explicitly mentioned.

      If there are any new team members, the process to be followed is explained so as to maintain quality standards of the organization. Clarity is given by the project lead if there exists any ambiguity in the process implementations.

      The kickoff is an enthusiasm-generator for the customer and displays a full summary of the project so far. By displaying a thorough knowledge of the goal and steps on how to reach it, the customer gains confidence in the team's ability to deliver the work.

      Steps
      Prepare presentation for project team
      Prepare a presentation containing the topics stated in  the software development plan.
      Plan kick-off meeting
      Plan the meeting to kick-off the project with all project members and the customer.
      Perform the kick-off meeting
      Inform all project members and the customer in a meeting.
      Ensure understanding from project members
      Ensure that all project members understand the project.
