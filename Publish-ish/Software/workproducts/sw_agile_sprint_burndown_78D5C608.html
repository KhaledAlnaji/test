Description
Main Description

A Sprint Burndown shows all remaining work of the so called the sprint backlog. This sprint backlog contains all work for the current sprint as committed by the team. Usually it is displayed as remaining work in ideal hours for each work day. The Sprint Burndown is updated on a daily basis by the team - often before the stand-up meeting of the next work day.

Like in the Product Burndown there is a working day zero to show the initially planned work before the first work day is finished. The purpose of the Sprint Burndown is to show the team if all planned tasks can be finished till the end of the sprint.
