Main Description
The Release planning is an high level plan created by customer / product owner and the development team considering the Epic priorities, but also the development risks and team velocity. It provides a high level overview of which epics can be delivered in which sprint.

If the development team's velocity is not known, an estimate must be provided for it, and the release plan must be understood to be less precise for the first few iterations, until a reliable velocity number can be derived.
