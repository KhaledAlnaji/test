Main Description

The Customer Software Requirements is the complete collection of all material that holds inputs to the Software Requirement Specification. Ideally this is a Software Requirements document provided by the customer or other stakeholders that is already specific for software development and can be imported as input requirements into the requirement management tooling.
However it is very common that this collection consists of various materials and various formats.

These requirements could come as written documents, pictures, models, data tables or any other format that holds information on what is required from software development.
