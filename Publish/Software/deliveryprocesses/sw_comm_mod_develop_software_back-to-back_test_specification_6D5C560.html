Main Description

1. Introduction
The generated code will always be slightly different from the Software Component Model. To see if the behaviour of the generated code is acceptable the tests of the the Model test Specification and the Software Component Model needs to be analysed to determine how the tests needs to be updated and which tolerances are acceptable.

2. Developping the Back-to-back test specification
The generated code should perform in the same way as the Software Component Model with some acceptable tolerances. Because the Model test specification is used to test the Software Component Model, it can be used as the basis for the Back-to-back test specification.
An analyses of the Software Component Model can aid in defining the tolerances allowed on the generated code. These tolerances in combination with the Model test specification define the test cases which are linked to the code for traceability.

Steps
Analyse Software Component Model
Determine acceptable tolerances for the generated code.
Define Back-to-back test specification
Document all tests in the Back-to-back test specification according the template.
Review Back-to-back test specification
Perform a peer review
Ensure Traceability
Establish traceability from the code to back-to-back test cases.
