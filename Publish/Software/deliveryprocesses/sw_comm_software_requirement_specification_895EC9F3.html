Main Description
The Software Requirement Speficitation is the document, or collections of documents, that holds all requirements that must be implemented during the project. This specification is the main outcome of the Develop Software Requirement Specification task.

The specification is documented in the requirement management tool. Each requirement is written as a single entity, each with individual parameters such as priority, status and validation criteria and is traced to its origins (for example to the Customer Software Requirements). Later in the development cycle, each requirement is additionally traced to one or more validation test cases in the Software Validation Test Specification and to downward documents such as the Software Architecture Design and to the Software Detailed Design or Software Model.
