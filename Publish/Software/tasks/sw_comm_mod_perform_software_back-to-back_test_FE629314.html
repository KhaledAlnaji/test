Task: Perform Software Back-to-Back Test
The creation and performance of the Back-to-back tests
Disciplines: All Software Tasks, ENG.6 - Software Construction
Purpose
To create and execute Software back-to-back tests to define if the generated code is acceptable.
Relationships
RolesPrimary Performer:
  • Software Tester
Additional Performers:
InputsMandatory:
  • Generated Code
  • Software Back-to-Back Test Specification
  • Software Master Test Plan
  • Software Model Test Result
    		Optional:
    • Software Model
      Outputs
      • Software Back-to-Back Test Result
        Main Description

        1. Introduction
        To determine if the behaviour of the generated code is acceptable, Software back-to-back tests need to be created and executed. The results will be compared to the Software Model Test results and documented.

        2. Creation and execution
        The creation of the Software back-to-back tests will be based on the software back-to-back specification and the software master test plan. Before the test will be executed a review needs to take place.

        3. Gathering results
        The results from the Software back-to-back tests will be gathered and analysed to determine if the generated code is acceptable. The results will be documented in the Software back-to-back test result specification.

        Steps
        Create Back-to-back tests
        Create the Back-to-back tests according the Back-to-back Test Specification
        Review Back-to-back tests
        Perform a peer review
        Perform software back-to-back tests
        Execute all back to back tests on the generated code.
        Gather test results
        Gather and document all test results into a Software Back-to-Back Test Result specification.
