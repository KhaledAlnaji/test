Work Product Slot: Demonstrated Software
The sum of all the Product Backlog items completed during a sprint and all previous sprints after demonstration.
Domains: All Software Work Products, MAN.3 - Project Managament
Purpose
A potentially shippable product demonstrated in the Sprint review meeting on which can be decided to accept it or not by the product owner.

Main Description

The Sprint Release is the sum of all the Product Backlog items completed during a sprint and all previous sprints. At the end of a sprint, the all backlog items must be done according to the Scrum Team's criteria called Definition of Done as stated in the Software development plan.  If this is not the case, the item will not be part of the Sprint Release.

The Sprint release will be demonstrated to the Product owner and stakeholders. Resulting in the demonstrated software. Based on the demonstration the product owner will decide if the demonstrated software will be accepted or not.

Key Considerations
The demonstrated software must be in a usable condition regardless of whether the Product Owner decides to actually release it.
