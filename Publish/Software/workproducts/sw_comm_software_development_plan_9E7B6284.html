Description
Main Description

1. Introduction
The Software Development Plan is created during the Project Start-up phase and needs to be updated continuously. The Software Development Plan is a source containing the latest information regarding  the project and its process. Changes in information that are relevant for the development team shall be communicated during team meetings.

2. Development plan
The Software development plan defines and coordinates all subsidiary plans  and integrates them into a comprehensive project plan. These subsidiary plans may be included into the development plan.

It defines the process how the project shall determine its scope and manages its requirements, defines the scheduling methodology, the accepted acuracy and tooling to be used in the development of the project schedule model.
It also will describe how the project costs will be planned, structured and controlled and how the communication will be structured, monitored and controlled.
